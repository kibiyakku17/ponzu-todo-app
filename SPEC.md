# ぽんずTodo 仕様書（v3.3）

- リポジトリ：`ponzu-todo-app`
- 公開URL：https://kibiyakku17.github.io/ponzu-todo-app/
- 構成：単一 `index.html` のPWA（ビルド不要）。データはすべて `localStorage`（この端末内のみ）。
- デザイン：ベージュ背景＋セージ/コーラル/スカイ/ラベンダー/アンバー/ローズのアクセント、Zen Maru Gothic（見出し）＋Zen Kaku Gothic New（本文）。
- コンセプト：**考えるのはAI。やるのはあなた。** AIで整理したリストを、コピペだけで登録できるTodoアプリ。

> バージョン履歴（要点）
> - **v3**：ホーム/管理の2画面分離、「今日だけ」タスク、種別/時間帯/カテゴリタグの一括投入
> - **v3.1**：AIコピペTodoのUI改善（ボタン名「AIリストを追加」、タグライン、ウェルカムカード、初回チュートリアル）
> - **v3.2**：ごみ出し設定のAIコピペ化・初期値を未設定に
> - **v3.3**：スケジュールタスク（頻度＋曜日＋週番号指定）・インポートタグ拡張・現状出力・データリセット

## 1. 全体構造（2画面＋下部タブ）

画面下部に固定ナビ（`position:fixed`）があり、2つの画面を切り替える。

| タブ | 役割 |
|---|---|
| 🏠 **ホーム** | 今日やることを見る・チェックする画面（追加/編集はしない） |
| ⚙️ **管理** | Todo・カテゴリ・ごみ出し・データ管理の追加/編集/削除を行う画面 |

「種別」（今日だけ/毎日/スケジュール）の切り替えは管理ページ内のタブで行う。ホームは種別を切り替えず、該当する全セクションを同時に縦表示する。

## 2. ホーム画面

- **ヘッダー**：アプリ名「ぽんずTodo」／今日の日付・曜日／タグライン「考えるのはAI。やるのはあなた。」／肉球メーター（進捗%）／一言メッセージ（スケジュール件数があれば「今日はN件のスケジュールタスクがあります。」）
- **ごみ出しバナー**：曜日に設定があれば自動表示。未設定の曜日は「今日のごみ出しはありません」表示。⚙️タップで管理ページの「ごみ出し」へ遷移
- **セクション**（データがある場合のみ、上から順に縦表示）：
  - 📌 **今日だけ**：対象日が今日 かつ 未完了 のタスク
  - 🔁 **毎日**：時間帯別（🌅朝／☀️昼休み／💻仕事中／🌇仕事終わり／🕐時間未設定）
  - 🗓 **スケジュール**：`isScheduledToday()` で今日に該当するタスクのみ表示（曜日・週番号が一致するもの）
- ホームは**チェック専用**（個別の追加/編集ボタンは置かない）
- 各セクション見出しに **−/＋ボタン**があり、完了済みの表示/非表示を切り替えられる

### チェックの挙動
- **毎日**：チェック状態は `daily_YYYY-MM-DD` キーで保存。完了済みは取り消し可能。
- **スケジュール**：チェック状態は `scheduled_YYYY-MM-DD` キーで保存。完了済みは取り消し可能。
- **今日だけ**：チェックすると `completed=true` になり**即座に非表示**。取り消し手段なし（単発タスクのため）。

### ウェルカムカード（Todo0件/初回/全削除直後）
- 「🐾 AIと一緒にTodoを作ろう」＋「📋 AIリストを追加を開く」ボタン
- Todoが1件でも登録されると非表示。0件に戻れば再表示

## 3. タスク種別と表示ロジック

### 今日だけ（`today`）
- 対象日が今日 かつ `completed=false` のものをホームに表示
- 対象日が過去で未完了（期限切れ）は管理ページ一覧には残るがホームからは消える
- データ：`todo_items_today_v3`

### 毎日（`daily`）
- 全件をホームに表示（時間帯でグループ化）
- 時間帯フィールド：`morning` / `lunch` / `work` / `evening` / `anytime`
- データ：`todo_items_daily_v3`

### スケジュール（`scheduled`）— v3.3追加
毎週・隔週・月1回などの不定期タスクを曜日＋週番号で管理する。

**表示判定ロジック（`isScheduledToday(item, today)`）：**
1. `item.schedDow !== today.dow` → 表示しない（曜日不一致）
2. `schedFreq === 'weekly'` → 表示する（毎週）
3. `schedFreq === 'biweekly'` or `'monthly'` → `schedWeekNums` の各 `n` について、その月の第n曜日の日付と今日の日付が一致するか確認

**第n曜日の計算（`nthWeekdayOfMonth(y, m, dow, n)`）：**
```
その月1日の曜日から最初の dow 曜日を求め、+(n-1)*7 で第n回目の日付を返す
```

**スケジュールタスクのデータ構造：**
| フィールド | 型 | 説明 |
|---|---|---|
| `id` | string | ユニークID |
| `main` | string | タスク名 |
| `note` | string | メモ（省略可、空文字） |
| `categoryId` | string | カテゴリID |
| `schedFreq` | `'weekly'`\|`'biweekly'`\|`'monthly'` | 頻度 |
| `schedDow` | 0–6 | 曜日（0=日〜6=土） |
| `schedWeekNums` | number[] | 対象となる週番号（weeklyは `[]`） |

データ：`todo_items_scheduled_v4`

## 4. 管理ページ

上部の横スクロールタブで以下を切り替える：

`📌今日だけ` / `🔁毎日` / `🗓スケジュール` / `🏷️カテゴリ` / `🗑️ごみ出し` / `⚙️データ`

### Todo種別タブ（今日だけ/毎日）
- 操作行：**＋個別追加** ／ **📋 AIリストを追加** ／ **チェックをリセット**（今日だけ以外）
- カテゴリ別のセクションでタスク一覧を表示。✏️ で編集
- 空状態は「📋 AIリストを追加」ボタンを表示

### スケジュールタブ
- **＋個別追加** ボタンでスケジュールタスクを1件ずつ追加
- カテゴリ別（カテゴリ未設定のタスクは「孤立アイテム」として `#888B7E` 色で表示）
- 各タスクに `fmtSched(item)` で頻度表示（例：「毎週月曜」「隔週 第2・第4日曜」「月1 第1日曜」）

### 個別追加・編集モーダル（スケジュール）
- フィールド：内容／メモ／カテゴリ／頻度（毎週/隔週/月1）／曜日／週番号（隔週・月1のみ）
- **モーダルは背景タップ・Escapeでは閉じない**（誤操作防止）

## 5. インポート/エクスポート フォーマット仕様

> ★ エクスポート（現状出力）とインポート（AIリストを追加）のフォーマットは**完全一致**。
> 出力したテキストをそのままAIに渡して編集し、貼り戻せる（ロスレス往復）。

### 基本ルール

| ルール | 説明 |
|---|---|
| 1行＝1タスク | 空行は無視 |
| 箇条書き記号除去 | `-` `・` `*` `①` `1.` `- [ ]` `□` 等は自動除去 |
| セクションタグ（単独行） | `[今日だけ]` `[今日だけ 6/20]` `[毎日]` |
| 時間帯タグ（単独行） | `[朝]` `[昼休み]` `[仕事中]` `[仕事終わり]` ※毎日セクション内のみ有効 |
| カテゴリタグ（単独行） | `[カテゴリ名]`（種別/時間帯/スケジュール以外のブラケット） |
| スケジュールタグ（インライン） | `[毎週 曜日]`、`[隔週 曜日 週番号...]`、`[月1 曜日 週番号]`＋同一行にタスク名 |
| 曜日指定 | `日` / `月` / `火` / `水` / `木` / `金` / `土`（1文字） |
| メモ | タスク行の**次の行を2スペースインデント**で記述 |

### スケジュールタグ詳細

| タグ | 意味 | 例 |
|---|---|---|
| `[毎週 曜日]` | 毎週その曜日 | `[毎週 月] 掃除機` |
| `[隔週 曜日 n...]` | 第n週のその曜日（複数可） | `[隔週 日 2 4] フローリング拭き` |
| `[月1 曜日 n]` | 月に1回、第n週のその曜日 | `[月1 日 1] シンク掃除` |

スケジュールタグは**インライン形式**（タグ＋タスク名を同一行に書く）。カテゴリタグは直前の `[カテゴリ名]` 単独行が引き継がれる。

### フォーマット例（エクスポート出力と同一）

```
[今日だけ]
[生活管理]
病院の予約電話
  午後2時以降にかける

[毎日]
[朝]
[健康]
ストレッチをする
[仕事中]
[仕事]
メールチェック
  重要フラグのみ返信

[家事]
[毎週 月] 掃除機
[毎週 土] シーツ・枕カバー交換
[毎週 日] お風呂掃除
[隔週 日 2 4] フローリングを拭く
  念入りに
[月1 日 1] シンク&コンロ掃除
[ぽんず]
[毎週 水] ぽんずのトイレ周り念入り掃除
[月1 日 2] ぽんずの爪・肉球確認
```

### インポート時のプレビュー
- 各タスク行に **カテゴリセレクト** が付き、個別に修正可能
- スケジュールタスクは頻度固定のため、freq/time セレクトのかわりに `🗓 毎週月曜` のようなバッジを表示（編集不可）
- 存在しないカテゴリは「✨新しく作るカテゴリ」として絵文字を指定でき、保存時に自動生成

## 6. 現状出力（AIレビュー用エクスポート）

管理ページ → **⚙️データ** タブ → 「出力する」ボタン。

`buildExportText()` が全タスクを上記フォーマットで生成し、モーダルのテキストエリアに表示。

**出力順序：**
1. 今日だけタスク（`[今日だけ]` セクション）
2. 毎日タスク（`[毎日]` → 時間帯ごとに `[朝]` 等のタグ → カテゴリタグ → タスク）
3. スケジュールタスク（カテゴリタグ → `[毎週 曜日] タスク名` のインライン形式）

メモがある場合は次行に2スペースインデントで出力。この出力をそのままインポートに貼り戻せる（ロスレス）。

「📋 コピー」ボタンでクリップボードにコピー可能。

---

## 7. データリセット

管理ページ → **⚙️データ** タブ → 「全タスクを削除する」ボタン。

- 確認ダイアログ表示後、以下のlocalStorageキーを削除：
  - `todo_items_today_v3` / `todo_items_daily_v3` / `todo_items_weekly_v3` / `todo_items_monthly_v3` / `todo_items_scheduled_v4`
  - `daily_YYYY-MM-DD` / `scheduled_YYYY-MM-DD`（全日付分）
- カテゴリ（`todo_categories_v3`）とごみ出し設定（`trash_schedule_v3`）は**削除しない**
- リセット後は空の状態になるのでウェルカムカードが表示され、「📋 AIリストを追加」からインポートできる

---

## 8. 運用フロー：ラウンドトリップ

**「出力→AIで見直し→リセット→入れ直し」** の流れで定期的にタスクを整理できる。

```
1. 管理→⚙️データ→「出力する」
      ↓ テキストをコピー
2. AIチャット（Claude/ChatGPT）に貼り付け
   「このタスクリストを見直して、不要なものを削除・追加・整理して」
      ↓ AIが編集したテキストを受け取る
3. 管理→⚙️データ→「全タスクを削除する」
      ↓ 全タスクをリセット
4. 管理→（任意のタブ）→「📋 AIリストを追加」
      ↓ AIの出力テキストをそのまま貼り付けて「追加する」
5. 完了
```

## 9. データ構造（localStorage）

| キー | 内容 |
|---|---|
| `todo_categories_v3` | カテゴリ一覧 `{id, name, emoji, color, soft}` |
| `todo_items_today_v3` | 今日だけタスク `{id, main, note, targetDate, categoryId, completed}` |
| `todo_items_daily_v3` | 毎日タスク `{id, main, note, time, categoryId}` |
| `todo_items_weekly_v3` | 旧週タスク（廃止UI・データのみ残存） `{id, main, note, categoryId}` |
| `todo_items_monthly_v3` | 旧月タスク（廃止UI・データのみ残存） `{id, main, note, categoryId}` |
| `todo_items_scheduled_v4` | スケジュールタスク `{id, main, note, categoryId, schedFreq, schedDow, schedWeekNums}` |
| `daily_YYYY-MM-DD` | 毎日タスクのチェック状態（`{タスクid: true/false}` のオブジェクト） |
| `scheduled_YYYY-MM-DD` | スケジュールタスクのチェック状態（同上） |
| `trash_schedule_v3` | ごみ出し設定（曜日index → `{label, mark, cls}` または null） |
| `todo_tutorial_seen_v3` | チュートリアル既読フラグ |
| `trash_hint_dismissed_v3` | ウェルカムのごみ出し案内スキップフラグ |

**スケジュールタスクの `schedFreq` 値：**
| 値 | 意味 |
|---|---|
| `'weekly'` | 毎週（`schedWeekNums` は `[]`） |
| `'biweekly'` | 隔週（`schedWeekNums` に対象週番号を複数指定、例 `[2, 4]`） |
| `'monthly'` | 月1回（`schedWeekNums` に週番号を1つ指定、例 `[1]`） |

## 10. マイグレーション方針

### v3→v3.3（旧週/月タスクのスケジュール化）

初回起動時（`todo_items_scheduled_v4` が未設定の場合のみ）、`initScheduledItems()` が自動実行される。

- `todo_items_weekly_v3` の各タスク → `schedFreq:'weekly', schedDow:6`（土曜週次）として `todo_items_scheduled_v4` に移行
- `todo_items_monthly_v3` の各タスク → `schedFreq:'monthly', schedDow:6, schedWeekNums:[1]`（第1土曜月次）として移行
- 旧キー（`_v3`）は削除せず残す（安全優先）
- スケジュールタスクが一件もない新規ユーザーは空の `[]` で初期化される（ハードコード投入なし）

## 11. カテゴリ

- 名前・絵文字・カラー（6色から選択）で作成。編集・削除も可能
- 削除時、紐づくTodoがあれば件数を確認ダイアログで通知（一緒に削除）
- 全種別で**共通のカテゴリ一覧**を使用
- **予約語はカテゴリ名にできない**：
  `今日だけ` `今日` `毎日` `週` `月` `週次` `月次`
  `朝` `昼` `昼休み` `仕事中` `仕事終わり` `夕方` `夜` `時間未設定`
  `毎週` `隔週` `月1`

## 12. ごみ出し設定（v3.2）

- **初期状態は全曜日「未設定」**。未設定の曜日はホームにバナーを表示しない
- 設定方法（管理ページ「🗑️ごみ出し」タブ）：
  - **方法A：AIに聞いて貼り付け**「🤖 AIに聞いて貼り付け」→ AIの回答を貼り付け → 曜日ごとにパース＆プレビュー → 保存
  - **方法B：手入力で編集**「⚙️ 手入力で編集」で曜日ごとに直接入力
- **パース仕様**：行頭の `[月]`〜`[日]` 曜日タグを認識し、後ろのテキストをその曜日の収集物に
- **表示色の自動判定**：燃やす系→coral / 缶・びん・ペット→sky / プラ→sage / 紙・資源→amber / その他→lavender

## 補足（実装上の判断）

- スケジュールタグはインライン形式（`[毎週 月] タスク名`）を採用。セクション見出し形式にすると「同じ曜日の複数タスク」で見出しが重複するため。
- 一括追加のプレビューは各行に編集用コントロールを並べる形（個別修正が確実なため）。スケジュールタスクはバッジ表示（頻度は変更不可）。
- 孤立アイテム（カテゴリが削除済みのタスク）はホーム・管理ページ両方で `#888B7E` 色で表示し、見えなくならないようにしている。
